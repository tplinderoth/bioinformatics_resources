## DIPLOID POPGEN VARIANT CALLING WORKFLOW ##

This documents step-by-step how to identify genetic variants from population NGS datasets involving diploid organisms.

The example code in this document pulls from multiple different studies to reflect the most up-to-date approaches 
for data processing.

Lines prefixed with '$' denote bash code.

## (2b) Map quality-controlled reads to the reference genome.
## The map_reads.sh script also marks residual PCR and optical duplicates based on mapping coordinates.

In order for the mapping script to set some of the SAM read group information (@RG) the names of the 
input files need to be formatted at follows:

<specimen_id>_<library_id>_qcpass_<lane number>_<forward or reverse read set>.fastq.gz

which should match regex

^[^_]+_[^_]+_qcpass_\d+_R[12].fastq.gz$

Below is an example formatted fastq name for forward reads for sample "CJ4773" from lane 1.

CJ4773_TLAWG10_qcpass_1_R1.fastq.gz

Below is an example formatted fastq name for reverse reads for sample "KEJBUH34" from lane 5.

KEJBUH34_TLAWG3_qcpass_5_R2.fastq.gz

If a sample was sequenced using only one lane, the fastq name must still include the lane number.

Optional:

At this point you can use local (or per contig) to genome-wide depth of coverage ratios to identify putative sex chromsomes 
if the the sex of some individuals is known.

Aternativley (or in addtion), you can generate coverage statistics for only the autosomes or sex chromosomes similar to above. This information 
can be used to assign sex based on the depth of coverage ratio between autosomes and sex chromosomes. Below is an example for 
Florida Scrub-Jays:

Calculate Z chromosome to autosome depth ratio for each individual using the zchr_depth_ratio.R script, which implements:

--- start R code ---

homolog_file = '/mnt/research/Fitz_Lab/ref/bird/FSJ_V3/FSJV3_convert.short.txt'
homolog = read.table(homolog_file,head=TRUE)
autosome = homolog$FSJV3_SCAFFOLD[which(homolog$ZEBRA_FINCH_CHR != "Z")]
zchr = homolog$FSJV3_SCAFFOLD[which(homolog$ZEBRA_FINCH_CHR == "Z")]

samples = read.table('/mnt/research/Fitz_Lab/projects/mosaic/mosiac_ids.txt',head=FALSE)

df = NULL
for (i in 1:nrow(samples)) {
        id = samples$V1[i]
        coverage = read.table(paste0('/mnt/research/Fitz_Lab/projects/mosaic/map/coverage/',id,'_coverage.txt'),head=FALSE)
        autoset = coverage[which(coverage$V1 %in% autosome),]
        zset = coverage[which(coverage$V1 %in% zchr),]
        len_total = sum(autoset$V3)
        autocov = sum(autoset$V7 * autoset$V3/len_total) # chromosome length weighted average
        zcov = zcov = zset$V7
        dratio = zcov/autocov # ratio of Z chr coverage to autosome coverage
        df = rbind(df, data.frame(SAMPLE = id, AUTOSOME_DEPTH = autocov, Z_DEPTH = zcov, DEPTH_RATIO = dratio))
}

# assign sex based on coverage ratio

df$NGS_SEX = NA
df$NGS_SEX[which(df$DEPTH_RATIO < 0.6)] = "Female"
df$NGS_SEX[which(is.na(df$NGS_SEX) == TRUE)] = "Male"

# write output

## (5a) Call variants with BCFtools and normalize INDELs.

BCFtools mpileup and call are used to identify variants. The BCFtools fill-tags plugin adds additional information 
that I use for downstream quality control. BCFtools norm left-aligns and normalizes INDELS with respect to the reference 
genome.

$ sbatch /mnt/research/Fitz_Lab/projects/atelopus/atelopus_wgs/variants/scripts/call_atelopus_variants_array.sh

# Use call P = 0.0028, which is the average nucleotide diversity for contemporary Atelopus varius (pi = 0.0031)

-f $REF \
-b $BAMS \
-R $SCAFFLIST \
-C 0 \
-d 1000000 \
-L 1000000 \
-q 20 \
-Q 20 \
--ns UNMAP,SECONDARY,QCFAIL,DUP \
-a FORMAT/AD,FORMAT/DP,FORMAT/QS,FORMAT/SP,FORMAT/SCR,INFO/AD,INFO/SCR \
-p \
-m 2 \
-F 0.05 \
-O u \
--threads 8 \
| bcftools call \
--ploidy 2 \
-a PV4,GQ,GP \
-m \
-P 0.0028 \
-G $GROUPFILE \
-O u \
--threads 8 \
| bcftools +fill-tags \
-O u \
--threads 8 \

--threads 8"

wait

Notes: The above variant identification code calls variants for chunks of the genome in parallel for efficiency (via the -R option). If the
genome is small and/or you have few samples parallelizing the calling may not be necessary. Here are some basic comments on the different calling 
parameters/options that I tend to use:

# mpileup options:

-C: This is a coefficient for downgrading the mapping quality. I typically do not use this (i.e. value of 0) unless a genome is particularly problematic 
for short read mapping (e.g. is highly paralogous or otherwise repetitive, the reference assembly is low quality) or reads are very short (e.g. < ~70bp 
and so are more difficult to accurately map).

-d: Limits the number of reads that are read per input file at a position. I set this to a high number to effectively disable capping the number of reads.
Even if not all reads are needed for accurate genotypig, I still like to use all of the information for calculating other statistics.

-L: Skips INDEL calling if the average per sample depth is above INT. I set this to a high value so at to always call putative INDELS irregardless of depth of coverage.

-q: Minimum map quality.

-Q: Minimum base quality.

--ns: Skips reads with any of the bits set. I only use high-quality, uniquely mapped reads that are not derived from PCR duplicates (which could otherwise
bias the genotype likelihood calculations, which assume 1-molecule:1-read).

-a: Additional annotations to output.

-p: Apply minimum number and fraction of gapped read cutoffs for identify putative INDELS at the per-sample level, which increases sensitivity for identifying 
potential indels.

-m: Minimum number of gapped reads for indel candidates.

-F: Minimum fraction of gapped reads for indel candidates.

# call options:

--ploidy: Ploidy of the analyzed sites (e.g. for autosomal sites in diploids this would be "2", for indivdiuals with heterogametic sex chromosomes this would be
"1" for the haploid regions of sex chromosomes, and would be "1" for mitochondrial sites). Note that using "2" for haploid organisms like bacteria is effective 
so that "genotypes" can be accurately called for colonies in which variants segregate.

-a: Output additional annotation information. I like to output statistics related to allelic biases (PV4),the genotype posterior probabilities (GP), and genotype quality (GQ) 
for subsequent quality control and analyses.

-m: Evokes using the multiallelic calling model, which I recommended (versus the older concensus caller, -c).

-P: Genetic diversity prior. This is the population-scaled mutation rate, and so estimates of Watterson's theta can be used here.

-G: This option can take a file that defines groups of samples for which a Hardy-Weinberg Equilibirum prior for the genotypes makes sense/is useful. If this option is not 
specified all individuals in the input are assumed to come from a single, panmictic popolation with genotype frequencies under HWE. If this option is set to "-", HWE is not 
used as a prior and single-sample genotype calling is performed (directly from the genotype likelhoods).

# fill-tags options:

-t: Adds additional annotations that I like to have in the VCF, e.g. about excess heterozygosity for identifying regions with confounded mapping, information about missing data, 
and alternate allele frequencies.

## (6h) Generate position files of quality-controlled sites.

# Extract positions of high-quality sites from the VCF (variable and monomorphic sites)

The following code extracts all sites from the VCF (and indexes the resulting position files for use with ANGSD) that meet the following quality criteria:
-FILTER="PASS".
-At least 90% of individuals from each group (this could be species, population, etc.) are covered by at least 2 reads. Note that the exact values are dataset specific.
-The site is either a SNP or monomorphic (excludes INDELs).
-SNPs are outside of INDELs.
-SNPs are biallelic.

The group representation filter helps to accurately quantify genetic structure in analyses involving multiple 
groups/populations/species etc.

$ sbatch /mnt/research/Fitz_Lab/projects/atelopus/atelopus_wgs/variants/scripts/extract_qc_sites_array.sh
implements:

--- start bash code ---

VCF="/mnt/gs21/scratch/lindero1/atelopus/wgs/variants/annotated/atelopus_annotated_${SLURM_ARRAY_TASK_ID}.vcf.gz"
SITES="/mnt/research/Fitz_Lab/projects/atelopus/atelopus_wgs/variants/vcf/qc_sites/sp_comparisons/atelopus_qc_sites_${SLURM_ARRAY_TASK_ID}.pos"

bcftools view -f "PASS" -i 'N_PASS(FMT/DP[0-10] > 1) > 9 && N_PASS(FMT/DP[11] > 1) > 0 && N_PASS(FMT/DP[12-13] > 1) > 1 && N_PASS(FMT/DP[14] > 1) > 0 && N_PASS(FMT/DP[15] > 1) > 0 && N_PASS(FMT/DP[16] > 
1) > 0 && N_PASS(FMT/DP[17-18] > 1) > 1 && N_PASS(FMT/DP[19] > 1) > 0 && (VT="." || VT="SNP")' -M 2 $VCF \
| bcftools query -f '%CHROM\t%POS\n' | uniq > $SITES

wait

angsd sites index $SITES

printf "\nfinished\n"

--- end bash code ---
